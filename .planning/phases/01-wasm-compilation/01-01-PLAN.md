<objective>
Consolidate WASI shim headers into a clean layered architecture with compile-time pointer width safety, and ensure the CMake/patch infrastructure correctly injects shims into the EdgeJS Emscripten build.

Purpose: The existing 44 shim headers are fragmented with overlapping responsibilities (flagged in CONCERNS.md). The wasm32 pointer width mismatch is the highest-risk silent corruption vector. This plan reorganizes shims and adds static assertions BEFORE attempting compilation, so errors are caught at compile time rather than as runtime corruption.

Output: Reorganized shim headers with pointer assertions, updated CMake toolchain, and updated EdgeJS patch file.
</objective>
